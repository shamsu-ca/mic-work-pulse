# WorkPulse Application Summary

## Overview
**WorkPulse** is a task management web application for internal staff operations at MIC (presumably an educational institution). It provides role-based access control with three user roles: Admin, Manager, and Assignee.

---

## Technology Stack
- **Frontend**: React 18 + Vite
- **Routing**: React Router v6
- **Backend**: Supabase (Auth + Database)
- **Styling**: Tailwind CSS with Material Symbols icons
- **State Management**: React Context (SupabaseDataContext)

---

## Page Details

### 1. LoginPage (`/login`)
- **UI**: Centered login card with gradient background, logo, email/password fields
- **Intention**: Authenticate users via Supabase auth

### 2. DashboardPage (`/`)
- **UI**: Conditionally renders AdminDashboard or AssigneeDashboard based on role
- **Intention**: Role-specific landing page with key metrics

### 3. StaffOverviewPage (`/staff`) — Admin only
- **UI**: Toggle between "Overview" and "Manage Staff" tabs
  - **Overview**: Staff cards with efficiency/workload metrics, expandable task list
  - **Manage**: Create user form + staff table with Edit/Reset PW actions
- **Intention**: Admin staff management and performance monitoring

### 4. AllTasksPage (`/tasks`)
- **UI**: Tabs (Active, Upcoming, Recurring, History), search bar, task table
- **Intention**: View all work items filtered by status

### 5. PlanningPage (`/planning`)
- **UI**: Left panel shows unassigned tasks in planning pool; right panel has allocation form
- **Intention**: Allocate unassigned tasks to staff with due dates

### 6. ProjectsEventsPage (`/projects-events`)
- **UI**: Type toggle (Projects/Events), mode toggle (Active/Saved Templates), expandable cards with task checklists
- **Intention**: Manage projects and events as containers for tasks

### 7. ReportsPage (`/reports`)
- **UI**: 
  - **Assignee**: Personal productivity score, task volume, overdue count, task history
  - **Admin**: KPI cards (global completion, completed, overdue, workload), staff leaderboard, activity log
- **Intention**: Performance analytics and reporting

### 8. MyTeamPage (`/my-team`) — Manager only
- **UI**: Left column shows team members with metrics; right panel shows selected member's tasks
- **Intention**: Managers view and assign tasks to their direct reports

---

## Role-Based Routes
| Role | Accessible Pages |
|------|------------------|
| Admin | Dashboard, Staff Overview, All Tasks, Planning, Projects/Events, Reports |
| Manager | Dashboard, All Tasks, Planning, Reports, My Team |
| Assignee | Dashboard, All Tasks, Planning, Reports |

---

## Key UI Components
- **AppLayout**: Fixed header with staff group toggle (Admin), sidebar navigation, mobile bottom nav, profile dropdown
- **CreateModal / CreateItemModal**: Forms to create tasks/projects/events
- **FilterBar**: Date range and staff group filtering (used in Staff Overview)
- **ProfileModal**: View/edit user profile

---

## Data Models (inferred from context)
- **profiles**: Staff members (id, name, email, role, staff_group, department, manager, avatar_url)
- **workItems**: Tasks (id, title, status, priority, assignee_id, container_id, expected_date, in_planning_pool, is_recurring)
- **containers**: Projects/Events (id, name, type, lead_id, progress, is_template)
